# API 参考文档质量评估规范（精简版）

> 仅保留高阶组件 API 参考文档检视相关要求。

---

## 一、质量评估维度

| 维度 | 要求 |
|------|------|
| **正确性** | 描述和功能实现一致，无前后矛盾或歧义 |
| **规范性** | 符合结构、格式、命名规范 |
| **完整性** | 提供开发者所需的全部必要信息 |
| **示例实用性** | 示例代码可编译、可运行、有效果图 |

---

## 二、API 参考文档质量要求

### 1. 一致性（正确性）

- 文档与代码一致，无超出、无遗漏
- 起始版本、入参、返回值与实际代码一致
- 接口名称无拼写错误

### 2. 描述完整性

**方法描述 8 要素：** 相似方法差异点、缺省配置、生效机制、系统限制、前提条件、注意事项、相关方法配合

**参数描述 9 要素：** 取值范围、单位、默认值、取值原则、生效机制、系统限制、前提条件、注意事项、参数配合

### 3. 示例代码要求

| 要求 | 说明 |
|------|------|
| 典型用法覆盖 | 贴近真实开发场景 |
| 效果图 | 涉及 UI 的示例**必须**提供效果示意图 |
| 可编译 | 片段代码直接拷贝进 DevEco Studio 可正常编译 |
| 可运行 | 经少量修改后能运行，与描述结果一致 |
| 关键代码注释 | 复杂逻辑需要注释帮助理解 |

| # | 检查项 | 归类 |

| 6 | Number 类型：边界值描述正确 | 规格正确性 |

| 8 | 默认值描述正确 | 规格正确性 |
