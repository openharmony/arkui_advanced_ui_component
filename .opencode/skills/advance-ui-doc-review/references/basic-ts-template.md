# ArkTS组件接口说明模板规范

## 总体写作说明

### 基础要求
- **文件命名**：ts-组件所属类名-组件名.md
  - 示例：基础组件Text → ts-basic-component-text.md
  - 示例：容器组件List → ts-container-list.md
- **上传路径**：docs/zh-cn/application-dev/reference/apis-xxx-kit/
- **图片路径**：docs/zh-cn/application-dev/reference/apis-xxx-kit/figures/

### 文档结构
1. 模块说明
2. 起始版本说明
3. 导入模块/使用说明
4. 子组件
5. 接口（属性、事件、方法、Class、枚举、Type）

## 文档标题规范

- **一级标题**：中文短语概括组件功能（如Button、Slider等可直接用英文）
- **版本说明**：使用引用语法`>`，放在模块描述之后
- 句式：`本模块首批接口从API version x开始支持。后续版本的新增接口，采用上角标单独标记。`

## 模块描述固定句式

```
（模块介绍，可选）xxx是xxx。
（功能描述，必选）提供xxx能力，包括xxx、xxx等。
（使用场景，可选）当需要xxx时，使用本模块xxx方法/本组件。
（使用建议或注意事项，可选）本模块可与xxx联合使用，以提升开发效率……
```

**示例：**
> Marquee
>
> 跑马灯组件，用于滚动展示一段单行文本，仅当文本内容宽度超过跑马灯组件宽度时滚动。
>
> **说明：**
> 该组件从API Version 8开始支持。

## 接口/属性/事件/方法说明格式

### 接口示例
```
Column(value?: {space?: string | number})

**卡片能力：** 从API version 9开始，该接口支持在ArkTS卡片中使用。
**原子化服务API：** 从API version 11开始，该接口支持在原子化服务中使用。
**系统能力：** SystemCapability.ArkUI.ArkUI.Full

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| space | string \| number | 否 | 纵向布局元素垂直方向间距。<br>默认值：0<br>单位：vp |
```

### 参数描述要求
- 参数的**含义与用途**
- **什么场景下使用**该参数
- **选取建议**、参数间**关联关系**
- **取值范围、单位、默认值、取值原则**
- 边界值涉及限制/异常时，需讲明具体场景
- 固定格式需给出格式样例（如URI）

## 常见问题

### 组件接口与API接口的区别
- 组件接口中的方法**不体现返回值**
- 其他要求同API接口中方法说明

### 尖括号显示问题
- `<>`可能被识别为标签导致显示失效
- 使用转义字符或`\>`保证正常显示
